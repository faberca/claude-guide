# Verification Protocol

Every task completion MUST include this report. Never just say "done" or "looks good".

| Item | Content |
|------|---------|
| Changed files | List of modified/created/deleted file paths |
| Verification | Command you ran + actual output (excerpt OK) |
| Evidence | Numbers: count, pass/fail, rows, bytes, etc. |
| Unresolved | "None" or specific description |
| Confidence | HIGH/MEDIUM/LOW + one-line reason |

## Banned phrases
- "It works well" (no evidence)
- "No issues" (what did you check?)
- "Done" (what exactly is done?)
