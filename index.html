Comparison

웹 AI vs Claude Code

웹 AI (ChatGPT / Gemini / Claude.ai)
  • 01 파일을 일일이 업로드해야 함
  • 02 결과물을 복사해서 수동으로 붙여넣기
  • 03 "이거 진짜 되는 건지" 직접 실행해봐야 앎
  • 04 실수하면 되돌리기 어려움
  • 05 대화 길어지면 앞에서 한 말 까먹음
  • 06 매번 "내 스타일은 이래" 다시 설명
  • 07 그럴듯하게 거짓말함 (환각)
  • == 사람이 일하고, AI가 조언
Claude Code (VSCode + Terminal)
  • 01 내 폴더 전체를 알아서 읽음 — 업로드 필요 없음
  • 02 파일을 직접 만들고 수정 — 복붙 안 해도 됨
  • 03 만든 걸 직접 실행하고 검증 — 되는지 확인까지
  • 04 실수하면 되돌려줌 — 버전 관리 자동
  • 05 대화 길어져도 핵심 유지 — 컨텍스트 자동 관리
  • 06 내 취향/규칙을 기억 — CLAUDE.md에 저장하면 매번 자동 적용
  • 07 거짓말 방지 — 추측 안 하고 파일 직접 읽어서 확인
  • == AI가 일하고, 사람이 방향만 잡아줌

Installation

원클릭 설치

PowerShell에 한 줄 붙여넣으면 Git, VSCode, Claude Code가 자동 설치됩니다.

Pro 구독 필요: Claude Code는 Pro ($20/월) 이상 계정이 필요합니다. 무료 플랜으로는 사용 불가.
PowerShell 여는 법
1 Win 키 누르기
2 powershell 입력
3 Enter (관리자 불필요)
PowerShell
irm https://claude-guide-sandy.vercel.app/install.ps1 | iex
한글이 깨져 보이면?

PowerShell 기본 인코딩이 한글을 못 읽는 경우입니다. 아래를 먼저 실행한 뒤 다시 시도하세요:

[Console]::OutputEncoding = [System.Text.Encoding]::UTF8

또는 한글 없이 작동하는 2줄 버전:

$f = "$env:TEMP\cc-install.ps1"; irm https://claude-guide-sandy.vercel.app/install.ps1 -OutFile $f; powershell -ExecutionPolicy Bypass -File $f

파일로 저장 후 실행하면 인코딩 문제가 거의 없습니다.

Getting Started

설치 완료, 이제 뭐 해?

3단계만 따라하세요.

1

작업할 폴더를 VSCode로 열기

탐색기에서 작업 폴더(예: 내 머미 프로젝트 폴더)를 찾아 우클릭 → "Code로 열기" 선택.

또는 VSCode 실행 후 Ctrl + K, O로 폴더 선택.

💡 폴더 없이 그냥 VSCode만 열면 Claude가 읽을 파일이 없어요. 반드시 폴더를 먼저 여세요.

2

VSCode 터미널 열기

VSCode에서 Ctrl + ` 누르면 하단에 터미널이 열립니다

3

Claude Code 실행

claude

브라우저가 안 열리면 c 키를 눌러 URL을 복사한 뒤 수동으로 열어주세요.

4

아무거나 시켜보기

코드베이스 질문하기

새 프로젝트에 투입됐을 때, 선배한테 물어보듯이 질문하세요.

이 프로젝트의 폴더 구조와 각 역할을 설명해줘
로그인 기능이 어떻게 구현되어 있는지 알려줘
Plan Mode: 탐색 → 계획 → 실행

큰 작업은 바로 코딩하지 말고, Plan Mode에서 먼저 파악하세요.

// Plan Mode (Shift+Tab)
src/auth 읽고 세션 관리가 어떻게 되는지 파악해줘
// Plan Mode
Google OAuth 추가하려면 어떤 파일을 바꿔야 해? 계획 세워줘
// Normal Mode
계획대로 구현해줘. 테스트도 작성하고 돌려봐.

주제 전환

큰 기능을 만들 때, Claude에게 먼저 질문하게 하세요. 놓치기 쉬운 엣지케이스를 잡아줍니다.

[기능 설명] 만들고 싶어. 먼저 나한테 질문해봐.
기술 구현, UX, 엣지케이스, 트레이드오프를 파고들어.
다 끝나면 SPEC.md로 정리해줘.
커스텀 스킬 만들기

반복 워크플로우를 /스킬이름으로 즉시 실행. .claude/skills/에 SKILL.md 파일 추가.

# .claude/skills/fix-issue/SKILL.md
---
name: fix-issue
description: GitHub 이슈 자동 수정
---
1. gh issue view $ARGUMENTS로 이슈 확인
2. 관련 파일 찾기
3. 수정 구현 + 테스트
4. PR 생성

사용: /fix-issue 1234

행동 규칙 (.claude/rules/)

Claude가 반복적으로 실수하는 패턴을 규칙으로 잡아두면 자동 교정됩니다. CLAUDE.md와 다른 점: rules/는 주제별로 분리 가능.

# .claude/rules/verification.md
작업 완료 시 반드시 보고:
- 변경 파일 목록
- 실행한 검증 명령 + 출력
- 수치 증거
- "잘 됩니다" 같은 증거 없는 보고 금지
서브에이전트 & 병렬 세션

복잡한 조사는 서브에이전트에게 위임. 메인 대화의 컨텍스트를 깨끗하게 유지.

서브에이전트를 써서 인증 시스템의 토큰 갱신 로직 조사해줘

Writer/Reviewer 패턴: 세션 A에서 구현 → 세션 B에서 리뷰 → A에서 피드백 반영

자동화 & 대규모 작업
# CI/스크립트에서 비대화형 실행
claude -p "API 엔드포인트 목록 만들어줘" --output-format json
# 2000개 파일 일괄 마이그레이션
for file in $(cat files.txt); do
claude -p "$file을 React에서 Vue로 변환" \
--allowedTools "Edit,Bash(git commit *)"
done

CLAUDE.md

프로젝트 규칙

터미널 3줄 이내 출력, 상세는 MD 저장. 검증 필수. 추측 금지. 심플하게 유지.

Claude가 매 대화 시작 시 자동으로 읽음

.claude/rules/ (4개)

행동 규칙

  • context-management — 출력 절약, 컨텍스트 보호
  • verification — 증거 기반 완료 보고 강제
  • code-accuracy — 팩트 확인 후 답변
  • self-improving — 실수 패턴 자동 규칙화
.claude/skills/ (3개)

즉시 사용 가능한 스킬

  • /commit-flow — 스마트 커밋 (변경 분석 → 메시지 생성 → 푸시 여부 확인)
  • /review — 코드 리뷰 (보안, 엣지케이스, 테스트 실행)
  • /explain — 파일/폴더를 쉬운 말로 설명
핵심 효과

설치만 하면 Claude가 알아서

  • 답변을 짧게 하고 길면 파일로 저장
  • 추측 대신 파일을 직접 읽고 확인
  • 작업 완료 시 증거와 함께 보고
  • 같은 실수를 반복하면 스스로 규칙 추가

CLAUDE.md에 뭐가 들어있나

스크립트가 설치하는 CLAUDE.md 하나에 아래가 전부 포함되어 있습니다. 개별 복사 필요 없음.

출력 규칙 — 터미널 3줄, 나머지 파일 저장
검증 강제 — 증거 없는 "완료" 금지
팩트체크 우선 — 추측 대신 파일 직접 확인
비판적 리뷰 — 자동 반론 3가지
실행 우선 — 3일 이월 시 자동 쪼개기
주도적 제안 — 빌드 후 자동화/커밋 제안
독립 검증 — 구현 후 재확인 필수
리서치 품질 — 3+ 소스, 원본 보존
컨텍스트 관리 — /clear, /compact 자동 안내
디자인 감각 — 트렌디 UI 자동 생성
자기개선 — 실수 패턴 자동 규칙화
Git 안전 — 확인 없이 푸시 금지

설치 후 첫 마디:

CLAUDE.md를 이 프로젝트에 맞게 수정해줘.
프로젝트 구조를 파악하고, 불필요한 규칙은 빼고, 이 프로젝트에 필요한 규칙을 추가해줘.

Claude가 프로젝트를 분석해서 CLAUDE.md를 자동으로 커스터마이즈합니다. 대화 필요 없음.

How to Use Well

잘 쓰는 핵심 개념

이것만 알면 Claude Code 퀄리티가 확 달라집니다.

1

컨텍스트 = Claude의 기억력

Claude는 대화 내용을 "컨텍스트 윈도우"라는 한정된 공간에 담아둡니다. 이게 가득 차면 앞에서 한 말을 까먹습니다. 웹 ChatGPT에서 대화 길어지면 멍청해지는 거랑 같은 원리.

/clear

주제 바뀌면 기억 리셋. 새 대화 시작하는 것과 같음. 가장 중요한 습관.

/compact

대화가 길어졌을 때 핵심만 압축. "요약본으로 이어가기" 느낌.

한 대화 = 한 주제

사이트 만들기 + 소설 교정을 한 대화에서 하면 둘 다 퀄리티 떨어짐.

2

CLAUDE.md = 매번 자동으로 읽는 설명서

프로젝트 폴더에 CLAUDE.md 파일을 넣어두면 매 대화 시작할 때 Claude가 자동으로 읽습니다. "나는 이렇게 해줘"를 매번 말할 필요 없음.

넣으면 좋은 것

"한국어로 답변해", "결과는 파일로 저장해", "추측하지 말고 확인해", "내 문체는 ~야"

효과

같은 말 반복 안 해도 됨. 세션 바뀌어도 규칙 유지. Claude가 점점 "내 비서"가 됨.

3

환각 방지 = "확인하고 말해"

AI는 모르는 것도 그럴듯하게 지어냅니다. Claude Code는 파일을 직접 읽을 수 있어서 훨씬 낫지만, 규칙을 안 잡으면 여전히 추측합니다.

CLAUDE.md에 한 줄

"추측하지 마. 파일 읽고 확인한 뒤에 답해." — 이거 하나로 거짓말 90% 감소.

"진짜야?" 습관

의심되면 "그거 어디서 확인한 거야?" 물어보기. Claude가 근거를 못 대면 지어낸 것.

검증 강제

"증거 없이 '됐다'고 하지 마" 규칙 넣으면 결과물+근거를 같이 보여줌.

4

파일 = 영구 기억

대화는 사라지지만 파일은 남습니다. 중요한 건 항상 파일로 저장시키세요. 다음 세션에서 그 파일을 읽으면 기억이 이어집니다.

나쁜 예

"조선시대 독약 알려줘" → 채팅으로 읽고 끝 → 다음에 또 물어야 함

좋은 예

"조선시대 독약 조사해서 research/독약.md로 저장해" → 영구 보존 → 다음에 바로 참조

Pitfalls

흔한 실수

잡탕 세션

한 대화에서 이것저것 → 컨텍스트가 쓰레기로 가득

해결: 주제 바뀌면 /clear

끝없는 수정

"아니 그게 아니라..." 3번 반복 → 실패 시도가 쌓여서 악화

해결: 2번 틀리면 /clear 후 처음부터

검증 없이 신뢰

그럴듯해 보여서 그냥 씀 → 엣지케이스에서 터짐

해결: "테스트 만들고 돌려봐" 한마디

CLAUDE.md 과잉

규칙 너무 많으면 Claude가 절반 무시

해결: "없으면 실수하나?" 기준으로 정리

소설·머미 작업할 때 특히 주의

원고 통째로 던지기

10만자를 한번에 → 컨텍스트 폭발 → 뒷부분 갈수록 앞 내용 까먹음

해결: 챕터별로 나눠서 파일 저장 → "3장 읽고 피드백해줘"

"전체 피드백 줘"

범위가 넓으면 "전반적으로 좋습니다" 같은 쓸모없는 답변

해결: 관점 지정 — "복선 누락 찾아줘", "캐릭터 말투 일관성 체크해줘"

지어낸 역사를 그대로 채택

"조선 정조 시대에 비소가 유행했다" — 그럴듯하지만 지어낸 것일 수 있음

해결: "출처 URL 같이 달아줘" 습관. 근거 못 대면 지어낸 것.

세계관이 대화에만 존재

설정을 채팅으로만 주고받음 → 세션 끝나면 전부 날아감

해결: 설정집을 MD 파일로 → "world/인물.md 읽고 이 설정 기준으로 써줘"

수정을 말로만 받기

"여기 이렇게 고치면 좋겠어요" → 내가 직접 옮겨야 함

해결: "파일에 직접 고쳐줘" — Claude Code는 파일 수정이 됨
